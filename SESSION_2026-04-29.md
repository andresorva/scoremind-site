## 3. CSS FIX FOR HERO PHONE (root cause)

Old CSS only had `.hero-phone { width: 360px; ... }` plus the global `img { display: block; max-width: 100% }`. The `<img>` had HTML attrs `width="1284" height="2778"`. In some browser/zoom combinations this caused the rendered height to exceed the aspect-ratio-correct value because nothing forced `height: auto` on the img.

New rule (line 199 in `index.html`):

```css
.hero-phone img {
  display: block;
  width: 100%;
  height: auto;
  object-fit: contain;
}
```

`height: auto` is the load-bearing fix — it tells the browser to compute height from `width × intrinsic-ratio` instead of using the HTML `height` attribute as a pixel value. `object-fit: contain` is a belt-and-suspenders guarantee against stretching even if the parent gets a height somehow.

**Verified after fix** via puppeteer `getBoundingClientRect()`: img renders at 360×779 (correct, matches native ratio 0.462). The `387×791` bounding rect at 1290px viewport is the 2° rotation widening the bounding box, NOT image stretching.
